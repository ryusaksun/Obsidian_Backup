根据提供的复习大纲和北大随机过程课件，排队论（Queueing Theory）是随机过程部分的主题 5。复习大纲明确指出重点在于定理加应用计算，特别是针对超星平台上的例子问题。

以下是排队论核心内容的详细总结：

## 基本概念与符号体系（Kendall 记号）

排队模型通常用 A/R/S/N 来描述：

- A（Arrival）：顾客到达的规律（通常是 M，代表泊松分布 Poisson/负指数分布 Markovian）。
- R（Service）：服务时间的分布（通常是 M，代表负指数分布）。
- S（Servers）：服务台（服务员）的数量（如 1 或 s）。
- N（Capacity）：系统容量限制（默认为 $\infty$，即无限队长；若为 N 则表示有限队长）。

## 核心指标与 Little 定律（重要考点）

在排队论计算中，主要关注以下四个均值指标：

1. $L$：在排队系统中的平均顾客数（队长，包含正在接受服务的）。
2. $L_Q$：在排队等候的平均顾客数（排队队长，不含正在接受服务的）。
3. $W$：顾客在系统中的平均逗留时间（逗留时间 = 等待时间加服务时间）。
4. $W_Q$：顾客在排队等候的平均时间（等待时间）。

**Little 定律（利特尔公式）**：这是排队论中最通用的定律，适用于各种到达和服务分布：

- $L = \lambda W$
- $L_Q = \lambda W_Q$
- 关系：$W = W_Q + \frac{1}{\mu}$（逗留时间 = 等待时间 + 平均服务时间）

## 典型模型：$M/M/1/\infty$（单服务台、无限容量）

这是最基础且考试最可能涉及计算的模型。

- 模型假设：
    - 到达过程：泊松过程，参数为 $\lambda$。
    - 服务时间：负指数分布，参数为 $\mu$（平均服务时间 $1/\mu$）。
    - 服务规则：先到先服务（FCFS）。
    - 稳态条件：服务强度 $\rho = \lambda / \mu < 1$（即服务速率必须快于到达速率，否则队列无限增长）。

- 稳态概率分布：系统中有 $n$ 个顾客的概率：

    $$p_n = (1-\rho)\rho^n, \quad n=0, 1, 2, \dots$$

    其中 $p_0 = 1 - \rho$ 是系统空闲的概率。

- 核心计算公式（必须熟记）：

    - 平均队长：$L = \frac{\rho}{1-\rho} = \frac{\lambda}{\mu - \lambda}$
    - 平均排队队长：$L_Q = \frac{\rho^2}{1-\rho} = L - \rho$
    - 平均逗留时间：$W = \frac{1}{\mu - \lambda}$
    - 平均等待时间：$W_Q = \frac{\rho}{\mu - \lambda} = \frac{\lambda}{\mu(\mu - \lambda)}$

## 其他模型（了解区别）

- $M/M/1/N$（有限容量）：
    - 系统最多容纳 $N$ 人。当系统满时，新来的顾客会离去（损失制）。
    - 关键点：有效到达率 $\lambda_e$ 会小于 $\lambda$，因为部分顾客流失了。需要先求出 $p_0$，再求 $p_N$（拒绝概率）。

- $M/M/s/\infty$（多服务台）：
    - 有 $s$ 个服务员。当顾客数 $n < s$ 时，不需要排队；当 $n \ge s$ 时才开始排队。

- 机器维修问题：
    - 属于有限源排队问题。顾客源（机器）总数是 $M$，坏了就去修。这部分有时也被归类在马尔可夫过程主题中考察。

## 解题步骤总结

根据复习大纲的提示定理加应用计算：

1. 识别模型：根据题目描述（由泊松流、负指数分布、服务台数量、是否有容量限制）判断是 M/M/1 还是 M/M/1/N 等。
2. 确定参数：找出 $\lambda$（单位时间到达数）和 $\mu$（单位时间服务数）。
3. 计算强度：计算 $\rho = \lambda / \mu$。
4. 套用公式：
    - 如果求概率（如系统空闲概率、恰有 n 人概率），用 $p_n$ 公式。
    - 如果求数量（队长），用 $L$ 或 $L_Q$。
    - 如果求时间（等待多久），用 $W$ 或 $W_Q$。
    - 灵活运用 Little 定律：算出其中一个（如 $L$），可以用 $L = \lambda W$ 快速推导 $W$。

**例题逻辑**：如果题目问顾客平均要等多久才能接受服务？，这是在问 $W_Q$。如果问顾客平均在店里停留多久？，这是在问 $W$。一定要分清是否包含服务时间。
