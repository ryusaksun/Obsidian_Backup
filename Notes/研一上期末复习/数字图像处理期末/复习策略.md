## 第一章：图像增强（空域）

老师强调: 本章重点是直方图均衡化计算和滤波原理。

- 直方图均衡化（★计算题高频）:
    - 考点: 给你一个简单的灰度分布表，让你计算均衡化后的新灰度级。
    - 步骤（死记）:
        - 统计各灰度级像素数量 $n_k$。
        - 计算概率（归一化）$p_r(r_k) = n_k / N$。
        - 计算累积概率 (CDF) $s_k = \sum p_r(r_j)$。
        - 计算新灰度值: $s_k \times (L-1)$ 并四舍五入取整。
    - PDF 位置: 第三章-图像增强final.pdf P.20-21 表格。
- 空域滤波（模板操作）:
    - 平滑滤波:
        - 均值滤波（线性）: 去噪，但模糊边缘。
        - 中值滤波（非线性）: 去除椒盐噪声效果最好，能保留边缘。
        - 计算方式: 给你一个 3x3 矩阵，让你算中心点的新值（排序取中间值或求平均）。
    - 锐化滤波:
        - Sobel 算子: 一阶微分。
        - 拉普拉斯算子: 二阶微分，对噪声敏感。
        - 重点: 背诵模板矩阵（如拉普拉斯中心是-4 或 8，Sobel 是-1,0,1 结构）。老师提到可能考算子包含的模板长什么样。
- 图像增强的概念:
    - 不是为了增加信息，而是为了改善视觉效果（去噪、变清晰）。

## 第二章：频域增强

老师强调: 概念多于计算，重点理解变换的意义和滤波器的对应关系。

- DFT 与 DCT:
    - DFT (傅里叶变换): <font color="#f79646">将图像转为频域。低频代表灰度变化慢（背景），高频代表灰度变化快（边缘、噪声）</font>。
    - DCT (离散余弦变换): <font color="#f79646">实数运算，能量集中在左上角（低频），常用于 JPEG 压缩</font>。
    - 公式: 了解二维 DFT 的定义公式形式。
- 频谱特征与卷积定理:
    - [[频谱图]]: 中心亮斑是直流分量(DC)，代表平均灰度。
    - 平移性: 图像平移，频谱幅度不变，相位改变（这一性质常考简答）。
    - 卷积定理: <font color="#f79646">空域卷积 = 频域乘积</font>。这是频域滤波的基础。
- 高通与低通滤波:
    - 低通滤波: 让低频通过 -> 平滑/模糊图像（去噪）。
    - 高通滤波: 让高频通过 -> 锐化图像（提取边缘）。
    - 对比: 要知道空域的均值滤波对应频域的低通；空域的锐化对应频域的高通。

## 第三章：边缘检测与图像分割

老师强调: 边缘检测（Canny）是重中之重，OTSU 大津法要懂原理。

- <font color="#ff0000">Canny 算子（★必背 5 步）</font>:
    - 这道题考简答题的概率极大，必须背下流程:
        - 高斯滤波（去噪）。
        - 计算梯度幅值和方向。
        - [[非极大值抑制 NMS]]（NMS，细化边缘）。
        - [[双阈值法]]（强边缘保留，弱边缘看情况）。
        - <font color="#f79646">滞后边界跟踪</font>（连接边缘）。
- [[霍夫变换]] (Hough Transform):
    - 原理: <font color="#f79646">利用点线对偶性，将图像空间中的点映射到参数空间</font>（$\rho, \theta$）。
    - 作用: 检测直线（可以连接断裂的边缘）。
    - 关键词: 参数空间、投票（累加器）、峰值检测（投票后找数值最大的那些格子）。
- [[大津法(OTSU)]]:
    - 原理: <font color="#f79646">基于最大类间方差。</font><font color="#f79646">遍历寻找最佳阈值，</font><font color="#f79646">使得前景和背景的方差差别最大</font>。
    - 不需要手算复杂公式，但要能用文字描述其核心思想。
- 其他分割方法:
    - [[区域生长]]: 种子点、生长准则（相似性）。
    - [[分水岭算法]]: 模拟地形浸水，解决粘连问题，但容易过分割。
    - [[GraphCut和GrabCut算法]]: 基于图论最小割，GrabCut 需要交互（画框）。

## 第四章：图像与视频压缩

老师强调: 计算题集中在这里（Huffman, RLE），视频压缩考概念。

- 无损压缩计算（★必考）:
    - 行程编码 (RLE): 例如 `aaaaa` 记为 `5a`。二维 RLE 通常用 Z 字形扫描。
    - 哈夫曼编码 (Huffman):
        - 步骤: 按概率排序 -> 合并最小的两个概率 -> 重复直到概率和为 1 -> 画树 -> 标 0/1 生成编码。
        - 考法: 给几个符号的概率，让你画树、写编码、算平均码长。
- 有损压缩 (JPEG):
    - 流程: 分块(8x8) -> DCT 变换 -> <font color="#f79646">量化（主要失真来源，去除高频）</font> -> 熵编码（Huffman/RLE）。
- 视频压缩:
    - 冗余: 时间冗余（相邻帧相似）、空间冗余。
    - 帧类型:
        - I 帧: 帧内编码（参考帧，像 JPEG）。
        - P 帧: 前向预测（参考前一帧）。
        - B 帧: 双向预测（参考前后帧，压缩率最高）。
    - 运动补偿: <font color="#f79646">不传整张图，只传运动矢量和预测残差（误差）</font>。
