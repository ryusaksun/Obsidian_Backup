以 `.` (点) 开头的文件在 Linux/Unix 系统中被称为隐藏文件 (Hidden Files) 或 点文件 (Dotfiles)。

## 1. 核心作用与意义

- 隐藏而非加密：它们并不是为了"加密"或"不让你看"，仅仅是为了在默认视图下保持目录整洁。
- 存储配置：绝大多数 `.config`、`.bashrc`、`.ssh` 这类文件都是软件的配置文件。如果不隐藏它们，你的主目录 (Home Directory) 会被几百个乱七八糟的配置文件塞满，很难找到你自己存的文档。

## 2. 如何操作它们

- 查看隐藏文件：
    殮通的 `ls` 看不到它们，必须加 `-a` (all) 选项。

```bash
ls -a
```

- 解释：列出所有文件，包括以 `.` 开头的。

- 创建隐藏文件：
    只需要在文件名前面加个点即可。

```bash
touch .my_secret_file
mkdir .my_hidden_dir
```

## 3. 两个特殊的点 (必知)

在每个目录下，你总会看到两个特殊的"点目录"：

- `.` (当前目录)：代表你现在所在的文件夹。
    - 常用场景：执行当前目录下的脚本 `./script.sh`。

- `..` (父目录)：代表上一级文件夹。
    - 常用场景：返回上一级 `cd ..`。

## 4. 历史趣闻

这就好比现在的"Bug 变成了 Feature"。
最早 Unix 的开发者在写 `ls` 命令时，为了不显示 `.` 和 `..` 这两个特殊目录，随手写了一行代码："如果文件名第一个字符是点，就不显示"。结果大家发现这个功能用来隐藏配置文件特别方便，于是这个约定就一直沿用至今了。

## 总结

- 看到 `.filename` -> 这是个配置文件/隐藏文件。
- 想看它们 -> 用 `ls -a`。
- 想执行当前目录的程序 -> 用 `./prog`。
